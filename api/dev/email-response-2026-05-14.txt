Dear Thomais,

Thank you for the feedback covering 4-11 May 2026. We have identified the following issues and applied the corresponding fixes.

1. Invalid Vehicle Identifier
   vehicleIdentifier in some requests contained non-ASCII characters,
   and in others the value used in POST /location did not match the
   value used in the preceding PUT /vehicles.
   Fix: vehicleIdentifier now strictly conforms to the spec pattern
   `^[A-Za-z0-9._~-]{1,40}$` and is used consistently across
   PUT /vehicles and POST /location for the same vehicle.

2. Data provided too early
   Driver Events were being submitted earlier than the 30-minute
   pre-pickup window.
   Fix: Driver Events are no longer submitted earlier than 30 minutes
   before the scheduled pickup time.

3. Pickup time in the past
   Location updates were sent beyond the acceptable post-pickup window.
   Fix: Drivers have been reminded to mark each job as completed at
   the end of the transfer.

4. Invalid Timestamp
   Payload format already matches the spec (UTC, `+00:00`, ISO 8601);
   the volume of errors indicates clock drift exceeding the 10-second
   tolerance.
   Fix: timestamp generation audited and unified; NTP synchronization
   verified on the host.
